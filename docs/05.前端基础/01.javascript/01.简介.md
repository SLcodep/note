## 问题1:数据类型

### 1.数据类型有哪些？
数据类型有8种，number、string、object、null、undefined、boolean、symbol、bigint

### 2.判断数据类型的方法有什么？
一共四种方法-typeof、instanceof、constructor、Object.prototype.toString.call
![数据类型](/js/数据类型.png)
判断数组优先用`Array.isArray()`

```javascript
Array.isArray([]) // true
Array.isArray({}) // false
Array.isArray(null) // false
Array.isArray(123) // false
```

### 3.typeof与instantof的区别？
- typeof是一个运算符，返回值是一个字符串，用来说明变量的数据类型。判断null返回的数据类型是object(历史bug)
- instanceof运算符用于指示一个变量是否属于某个对象的实例。返回值为布尔值。instanceof 主要的实现原理就是只要右边变量的 prototype 在左边变量的原型链上即可，返回true/false。判断null是无意义

###  4.instanceof操作符的实现原理与实现？
instanceof 运算符用于判断构造函数的 prototype 属性是否出现在对象的原型链中的任何位置。

## 问题2:为什么 0.1 + 0.2 不等于 0.3？
0.1 + 0.2 ≠ 0.3 的核心原因是JavaScript 使用**二进制浮点数**（IEEE 754 标准）存储小数，而 **0.1 和 0.2 这类十进制小数无法被二进制精确表示**，计算时因精度丢失产生了微小误差，最终导致结果不等于 0.3。

## 问题3:为什么 setTimeout(setInterval)不精确？如何解决？
**为什么不精确？**
1.js 是单线程的，执行任务需要先等待同步任务、微任务执行完成后，才会执行延时任务。
2.在嵌套超过 5 层，如果设置的 timeout 小于 4ms，则会强制将其设置为 4ms，即会导致至少 4ms 的延迟。
3.页面失活（页面在后台）会导致不准确。
**如何解决？**
用 `performance.now()` 做时间校准，通过 “递归 setTimeout + 时间补偿” 修正偏差，放弃 `setInterval`；

## 问题4:== 和 === 的区别？
`==` 宽松相等（相等）：`==` 会先尝试类型转换，将不同类型的值转换为同一类型后，再比较值是否相等。
`===` 严格相等（全等）：`===` 会同时比较值和数据类型，只有当两者完全一致时，结果才为 true，不会进行任何类型转换。如果类型不一致，直接返回 false。

**`==` 和`===`典型示例**
:::code-group
```js [==示例]
// 规则1：null/undefined 互等
null == undefined // true

// 规则2：数字 vs 字符串，字符串转为数字
123 == '123' // true

// 规则3：布尔值 vs 其他类型，布尔值转为数字再按其他规则比较
true == 1 // true（true 转 1）
false == '' // true（false 转 0，'' 转 0）

// 规则4：对象 vs 基本类型，对象先调用 valueOf() 或 toString() 转为基本类型，再比较
[1] == 1 // true（[1].valueOf() → [1]，再 toString() → '1'，转数字 1）

```
```js [===示例]
// 基本类型：类型不同直接 false
123 === '123' // false（数字 vs 字符串）
true === 1 // false（布尔 vs 数字）
null === undefined // false（不同类型）

// 基本类型：类型+值都相同 → true
123 === 123 // true
'abc' === 'abc' // true
null === null // true

// 引用类型：判断是否指向同一内存地址
[] === [] // false（两个不同的数组对象）

```
:::

## 问题5:如何理解 JS 的异步？
JS 是一门单线程的语言，这是**因为它运行在浏览器的渲染主线程中，而渲染主线程只有一个**。
而渲染主线程承担着诸多的工作，渲染页面、执行 JS 都在其中运行。
**如果使用同步的方式，就极有可能导致主线程产生阻塞**，从而导致消息队列中的很多其他任务无法得到执行。这样一来，一方面会导致繁忙的主线程白白的消耗时间，另一方面导致页面无法及时更新，给用户造成卡死现象。
所以浏览器采用异步的方式来避免。具体的做法是当某些任务发生时，比如计时器、网络、时间监听，主线程将任务交给其他线程去处理，自身立即结束任务的执行，转而执行后续代码。当其他线程完成时，将事先传递的回调函数包装成任务书，加入到消息队列的末尾排队，等待主线程调度执行。

## 问题6:null 和 undefined 的区别？
`null` 是表示 “**空值**” 的关键字，是主动声明的、有意义的空（比如表示变量本该指向对象但目前无指向）；
`undefined` 是表示 “未定义” 的原始值，是被动的、未初始化的状态（比如**变量声明未赋值**、函数无返回值时默认返回）

注意：`typeof null → "object"`是JS最初实现的bug。
- 若要精准判断 null，需用 === null
```javascript
function isNull(val) {
  return val === null; // 唯一精准的方式
}
```

## 问题7:let、const、var的区别？
`var` 是 ES5 的变量声明方式，存在函数级作用域、变量提升、可重复声明的问题；
`let` 和 `const` 是 ES6 新增的**块级作用域**声明方式，let 声明可变变量，const 声明只读常量（声明时必须赋值，且指向的内存地址不可变），二者均无变量提升、不可重复声明，还会形成暂时性死区。

- **全局声明的差异**
`var` 声明的全局变量会挂载到 window 对象上，易污染全局环境；
`let/const` 声明的全局变量不会挂载到 window，仅存在于全局作用域，更安全。

## 问题8:如果new一个箭头函数的会怎么样?
`new` 关键字无法作用于箭头函数，执行 new 箭头函数() 会直接抛出 TypeError 错误；
**核心原因**是箭头函数没有 prototype 原型对象，也没有 constructor 构造器，不具备 new 操作的底层执行条件（无法创建实例、绑定 this）。

:::tip
**new 操作的底层执行逻辑**
new 一个普通函数时，JS 引擎会执行以下步骤：
  - 1.创建一个空的新对象；
  - 2.将新对象的 __proto__ 指向函数的 prototype 原型对象；
  - 3.将函数的 this 绑定到这个新对象；
  - 4.执行函数体代码（若函数有返回值且是对象，返回该对象；否则返回新创建的对象）。

核心前提：**被 new 的函数必须有 prototype 属性，且能被绑定 this**
:::

## 问题9:请谈一下箭头函数与普通函数的区别？
箭头函数是 ES6 为简化函数写法、解决普通函数this动态绑定问题设计的语法糖，**核心区别在于：**
箭头函数无独立this（词法绑定）、无prototype/constructor、不能被new实例化、无arguments；
而普通函数有动态this、具备构造器特性、可实例化，功能更完整但写法繁琐。

**1.this绑定（最核心区别）**
- **普通函数**：this 是 “动态绑定”，指向调用该函数的对象，不同调用方式this不同，易踩坑；
- **箭头函数**：this 是 “词法绑定/静态绑定”，**直接继承定义时所在上下文的this**，永远无法修改（call/apply/bind也无效）。

::: details 💡箭头函数的this指向
**对象方法中的箭头函数（高频坑点）**
箭头函数作为对象方法时，this 不指向对象本身，而是**继承外层全局/函数的 this**
```javascript
const obj = {
  name: 'obj',
  // 箭头函数作为对象方法
  arrowFn: () => {
    console.log(this.name); // undefined（this指向全局，全局无name）
  },
  // 普通函数作为对象方法
  normalFn: function() {
    console.log(this.name); // 'obj'（this指向调用者obj）
  }
};

obj.arrowFn();  // undefined（坑点：新手易误以为this指向obj）
obj.normalFn(); // 'obj'

// 嵌套场景：对象方法内的箭头函数（正确用法）
const obj2 = {
  name: 'obj2',
  outerFn: function() {
    // 外层是普通函数，this指向obj2
    const innerArrow = () => {
      console.log(this.name); // 'obj2'（继承outerFn的this）
    };
    innerArrow();
  }
};
obj2.outerFn(); // 'obj2'（箭头函数的核心价值：继承外层this）
```

:::

## 问题10:请谈一下map和Object的区别？
`Map` 是 ES6 新增的键值对集合类型，专为 “键值映射” 场景设计，支持任意类型作为键、可迭代、有序且能精准控制键值数量
`Object` 是 JS 基础数据类型，本质是 “对象”（兼具哈希表特性），键仅支持字符串/符号类型、无序、会继承原型链属性，二者核心差异体现在键类型、有序性、迭代性和内存效率上。

## 问题11:什么是 DOM 和 BOM？
`DOM`（文档对象模型）是浏览器将**HTML/XML文档解析成的树形对象结构**，核心用于操作页面的元素、样式和内容；
`BOM`（浏览器对象模型）是**操作浏览器本身的对象集合**，核心用于控制浏览器窗口、导航、存储等非页面内容的功能。
二者都是浏览器提供的JS编程接口，DOM聚焦“页面内容”，BOM聚焦 “浏览器环境”。

## 问题11:ES6新特性?
ES6（ES2015）是 JavaScript 史上最核心的版本升级，新增了 **块级作用域、类、模块化、箭头函数、解构赋值、模板字符串、Set/Map 集合、Promise** 等核心特性，解决了 ES5 的诸多痛点（如无块级作用域、模块化缺失、异步编程繁琐），大幅提升了代码的可读性、可维护性和开发效率。

## 问题12:WeakMap 与 Map 的区别
1.WeakMap 的 key 必须是对象，而 Map 可以是任意类型
2.WeakMap 不支持迭代对象的方法
3.当 GC（垃圾回收）时，WeakMap数据会被回收（与 Map 的最大区别）

:::details 什么是强引用和弱引用？
弱引用是不能确保其引用的对象不会被垃圾回收器回收的引用，而强引用是确保其引用的对象不会被垃圾回收器回收的引用。
也就是说，**JavaScript 引擎在执行代码时，对象通过变量直接赋值形成的引用会被视为强引用**，垃圾回收器就不会回收这类对象；通过 `WeakMap` 和 `WeakSet` 建立的引用会被视为弱引用，弱引用不影响垃圾回收。
当一个变量被设置为 `null` 时，会断开该变量与原对象间的引用，该对象就会变成垃圾回收器的回收目标。
:::
