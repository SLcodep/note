## 基础相关概念
> 执行上下文属性:变量对象(Variable Object，VO)、作用域链(Scope Chain)、this
![](/js/this.png)

- 全局作用域里的 this 是 window，严格模式下是 undefined
- 函数的 this，看执行主体前有没有点，有点，前面是谁，函数里的 this 就是谁，没有点，函数的 this 就是 window，严格模式下是 undefined
- 自执行函数里的 this 是 window，严格模式下是 undefined
- 回调函数里的 this 一般情况下是 window
- 箭头函数没有 this，在箭头函数中使用的 this，是上级作用域或作用域链中找到的 this，直到找到全局的 window。
- 构造函数里的 this 是当前实例
- 实例原型上的公有方法里的 this 一般是当前实例
- 给元素绑定事件行为事件里的 this 就是当前被绑定的元素本身

## 1.默认绑定
**默认绑定通常是指函数独立调用**，不涉及其他绑定规则。非严格模式下，`this指向window`，严格模式下，this指向undefined。

### 对象内执行
foo虽然在obj的bar函数中，但foo函数仍然是独立运行的，foo中的this依旧指向window对象。
```js
a = 1;
function foo() {
    console.log(this.a); 
}
const obj = {
    a: 10,
    bar() {
        foo(); // 1
    }
}
obj.bar();
```
### 自执行函数

>默认情况下，自执行函数的this指向window。自执行函数只要执行到就会运行，并且只会运行一次，this指向window。

```js
a = 1;
(function(){
    console.log(this);
    console.log(this.a)
}())         //自执行this指向win
function bar() {
    b = 2;   //未声明的变量赋值, 自动挂载到全局对象
    (function(){
        console.log(this);
        console.log(this.b)
    }())
}
bar();       //默认绑定
```

## 2.隐式绑定
函数的调用是在某个对象上触发的，即调用位置存在上下文对象，通俗点说就是 **XXX.func()** 这种调用模式。如``

```js
//对象链式调用
const obj1 = {
    a: 1,
    obj2: {
        a: 2,
        foo(){
            console.log(this.a)
        }
    }
}
obj1.obj2.foo() // 2    谁调用函数，this 就指向谁
```

## 3.隐式绑定的丢失
隐式绑定是如函数调用，一般会有两种常见的丢失:
- 使用另一个变量作为函数别名，之后使用别名执行函数
- 将函数作为参数传递时会被隐式赋值
**隐式绑定丢失之后，this的指向会启用默认绑定**。

### 取函数别名
>JavaScript对于引用类型，其地址指针存放在栈内存中，真正的本体是存放在堆内存中的。
将obj.foo赋值给foo，就是将foo也指向了obj.foo所指向的堆内存，此后再执行foo，相当于直接执行的堆内存的函数，与obj无关，foo为默认绑定。笼统的记，只要fn前面什么都没有，肯定不是隐式绑定。

```js
a = 1
var obj = {
    a: 2,
    foo() {
        console.log(this.a)
    }
}
var foo = obj.foo;
obj.foo();
foo();      //与obj无关，已经指向obj.foo所指向的堆函数，为默认绑定
```

### 参数传递
JS中函数参数传递传递的是**函数的引用类型**。

>把 obj.foo 传给 doFoo 的 fn 参数时，只是把 foo 函数本身的引用传了过去，并没有附带 obj 的绑定关系。

```js
function foo() {
  console.log(this.a) // 关键：this指向由调用方式决定
}

function doFoo(fn) {
  console.log(this)   // 非严格模式下，doFoo独立调用 → this指向window
  fn()                // 独立调用fn（foo函数）→ this指向window
}

var obj = { a: 1, foo } // obj.foo 是 foo 函数的引用
var a = 2               // 全局变量a → 挂载到window，即window.a = 2

doFoo(obj.foo) // 输出：window → 2
```

## 箭头函数
在 ES6 中引入了箭头函数，**箭头函数没有自己的 this**。它的 this 取决于 **声明时所在作用域的 this**，而不是调用方式。
:::details 💡箭头函数的作用
ES6的新特性箭头函数,关于this的指向,有一条是**回调函数里的 this 一般情况下是 window**。下面的代码就是在`forEach`独立调用传入的回调函数,在非严格模式下，独立调用的函数 this 会指向全局对象。
```js
const person = {
  name: "小明",
  hobbies: ["篮球", "游泳"],
  showHobbies: function() {
    this.hobbies.forEach(function(hobby) {
      console.log(this.name + "喜欢" + hobby);
    });
  }
};

person.showHobbies();  
// 输出：undefined喜欢篮球
// 输出：undefined喜欢游泳
```
这时候就体现出箭头函数的好处，**箭头函数的this继承自外层作用域**。
```js
const person = {
  name: "小明",
  hobbies: ["篮球", "游泳"],
  showHobbies: function() {
    this.hobbies.forEach((hobby) => {
      console.log(this.name + "喜欢" + hobby);
    });
  }
};
person.showHobbies();  
// 输出：小明喜欢篮球
// 输出：小明喜欢游泳
```
