## Prisma

项目中**Prisma ORM** 的数据模型定义文件`schema.prisma`，用于描述 PostgreSQL 数据库的表结构、字段类型、关系、约束等信息。

Prisma 会基于这个文件自动生成类型安全的数据库客户端，让你可以用直观的方式操作数据库，无需手写原生 SQL。

| 模型名              | 对应业务含义                 | 核心字段 / 约束                                              |
| ------------------- | ---------------------------- | ------------------------------------------------------------ |
| `User`              | 用户表                       | 主键 `id`（自动生成 CUID）、唯一 `email`、关联账户 / 会话 / 聊天记录、创建 / 更新时间 |
| `Account`           | 第三方登录账户表（如 OAuth） | 关联 `User`（级联删除）、唯一约束 `[provider, providerAccountId]`（同一第三方账号唯一） |
| `Session`           | 用户会话表（登录态）         | 唯一 `sessionToken`、关联 `User`、过期时间 `expires`         |
| `VerificationToken` | 验证令牌表（如邮箱验证）     | 复合唯一约束 `[identifier, token]`                           |
| `ChatSession`       | 聊天会话表                   | 关联 `User`（可选，支持游客）、`guestId`（游客标识）、标题、是否置顶、关联消息 / 附件 |
| `Message`           | 聊天消息表                   | 关联 `ChatSession`（级联删除）、`role`（枚举）、内容、模型 ID、推理过程 |
| `Attachment`        | 聊天附件表                   | 关联 `ChatSession`（级联删除）、文件名、MIME 类型、文件大小、文本内容 |

## Auth.js

> Auth.js本质上是项目的认证和会话管理层，关注的是：**当前请求是谁发起的，他是否登录，他的登录状态是否可信且可持续**

**针对这个项目主要做的四件事：**

- 社交登录：GitHub / Google OAuth
- 登录会话：用户登录后如何持续识别
- 服务端鉴权：接口收到请求时如何知道当前用户是谁
- 与数据库联动：把用户、账号、会话这些认证数据持久化到 Prisma / PostgreSQL

Prisma负责数据怎么存，Auth.js负责谁在用这些数据





除此之外还有针对访客`GuestID`(访客ID)针对**未登录/匿名用户**的临时身份标识，本质是给「非认证用户」分配的唯一标识符。

**GuestID 的核心特征**

- 匿名性：不需要用户注册 / 登录，系统自动生成（通常是 UUID / 随机字符串）
- 临时性：能随会话结束（比如关闭浏览器）失效，或长期保留（比如存到本地缓存）
- 唯一性：每个匿名用户的 GuestID 不重复，能区分不同的匿名访问者

| 维度     | Auth 认证（登录用户）       | GuestID（匿名用户）     |
| -------- | --------------------------- | ----------------------- |
| 身份级别 | 高权限（可操作个人数据）    | 低权限（仅基础功能）    |
| 持久性   | 长期有效（可记住登录）      | 临时 / 短期有效         |
| 信息关联 | 绑定手机号 / 账号等实名信息 | 无实名信息，仅匿名标识  |
| 核心作用 | 精准身份 + 权限控制         | 匿名身份识别 + 基础追踪 |
